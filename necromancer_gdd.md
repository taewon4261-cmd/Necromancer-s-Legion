# 게임 기획서: 네크로맨서 군단 (Necromancer's Legion)

---

## 1. 프로젝트 개요 (Overview)

*   **게임 타이틀:** 네크로맨서 군단 (Necromancer's Legion - 가제)
*   **게임의 장르:** 탑다운 2D 로그라이트 서바이벌 (뱀파이어 서바이버 류)
*   **핵심 컨셉:** 본체는 적을 피해 도망치고, 쓰러진 적을 내 언데드 군단으로 자동 부활시켜 수백 마리가 맞붙는 대규모 자동 전투. 
*   **레퍼런스 게임 및 예상도:** Vampire Survivors(직관적 조작/레벨업), Thronefall(미니언 중심 전투). 어두운 픽셀 아트 스타일.

## 1-2. 핵심 게임 루프 및 성장 루프 (Core Loops)

1인 개발 뱀서류에서 가장 중요한 **재미의 순환 구조(Loop)**를 두 가지 톱니바퀴로 분리하여 설계합니다.

### 1. 핵심 전투 루프 (In-Game Combat Loop)
"어떻게 살아남으며 싸울 것인가"에 대한 초단기적(수 초~수 분 반복) 액션 구조입니다.
*   **회피 및 유인:** 플레이어는 몰려오는 적을 피해 맵을 크게 돌며 도망치고 적들을 한 곳으로 뭉치게 만듭니다.
*   **첫 킬 (씨앗 발아):** 본체의 기본 자동 공격(회전하는 낫 등)이 가장 약한 앞선의 적을 쓰러뜨립니다.
*   **군단 부활 (이 게임만의 핵심 차별점):** 적이 죽은 위치에서 즉시 나의 '언데드 미니언'이 통제가 불가능한 상태로 부활해 적들을 향해 알아서 돌진합니다.
*   **전투 페이즈 전환:** 플레이어 대신 미니언들이 고기방패 및 딜러 역할을 하며, 전장의 분위기가 '단순 도망'에서 '대규모 난전'으로 자동 전환됩니다.
*   **루팅 및 반복:** 미니언과 적이 싸우며 쏟아내는 경험치 보석(영혼)을 플레이어가 주워 담으며 생존을 이어나갑니다.

### 2. 핵심 성장 루프 (Growth & Meta-Progression Loop)
"어떻게 피지컬적인 한계를 극복할 것인가"에 대한 중장기적 플레이 동기부여 구조입니다.

**A. 인게임 판당 성장 (In-Game Level Up)**
*   **경험치 획득:** 전투 루프에서 떨어진 보석을 수집하면 화면 하단의 경험치 바가 서서히 찹니다.
*   **시간 정지 및 선택:** 경험치 바가 꽉 차면 레벨업 UI가 뜨고 게임이 완전히 멈춤니다. 3~4개의 무작위 특성(예: '미니언 이동 속도 20% 증가', '독성 연속 부여') 중 하나를 고릅니다.
*   **파워 펌핑:** 선택 직후 내 군단 전체가 일제히 강해지며, 곧이어 도래할 더 빽빽한 다음 타임 웨이브를 간신히 버텨낼 맷집을 얻습니다.

**B. 메타 성장 루프 (Out-Game 영구 업그레이드)**
*   **재화 누적:** 게임 내내 처치해 얻은 폐지(골드/뼈 등 특수 재화)는 게임 오버 시에도 잃지 않고 타이틀 로비로 가져옵니다.
*   **영구 스탯 투자:** 로비 화면의 상점에서 골드를 소모해 '초기 플레이어 이동 속도 5% 상승', '시작하자마자 미니언 5마리 기본 제공', '경험치 획득 자석 범위 영원히 증가' 등 패시브 스탯을 구매합니다.
*   **재도전 동기 형성:** "아까는 방패 든 기사들한테 5분 만에 죽었는데, 방금 기본 이속을 올렸으니까 이번엔 무조건 10분까지 버틸 수 있겠다!"라는 마음으로 리트라이(Re-try) 버튼을 누르게 만듭니다.

## 2. 씬(Scene) 설계 및 하이어라키 구조

1인 개발의 효율성을 위해 **단 2개의 씬(Scene)**으로 구성합니다.

### 2-1. `TitleScene` (타이틀 및 로비)
*   **역할:** 게임 시작, 설정, 영구 강화(업그레이드) 상점.
*   **하이어라키 (Hierarchy) 구조:**
    *   `[Managers]` -> TitleManager, SoundManager(DontDestroyOnLoad)
    *   `[UI_Root]` (Canvas)
        *   `Panel_Title` -> 게임 로고 이미지, "Start", "Upgrade", "Setting" 버튼
        *   `Panel_Upgrade` -> 시작 시 영구 능력치(이속, 시작 골드 등)를 올리는 로비 상점 창
        *   `Panel_Setting` -> 볼륨 조절 창
    *   `EventSystem`

### 2-2. `GameScene` (메인 인게임)
*   **역할:** 실제 웨이브 생존 전투, 레벨업, 결과(죽음/승리) 처리.
*   **하이어라키 (Hierarchy) 구조:**
    *   `[Managers]`
        *   `GameManager` (전반적인 State 관장)
        *   `WaveManager` (시간 및 몬스터 스폰 테이블 관리)
        *   `ObjectPoolManager` (수백 마리의 몬스터/이펙트를 재활용하기 위한 풀링 관리)
        *   `ExpManager` (경험치 보석 테이블 관리)
    *   `[Environment]`
        *   `Tilemap_Floor` (바닥 무한 타일맵)
        *   `Tilemap_Obstacle` (가장자리 막힘이나 장애물 - 필요시)
    *   `[Entities]`
        *   `Player` (네크로맨서 본체 오브젝트) -> 하위에 MainCamera를 달거나 가상 카메라(Cinemachine)로 추적
        *   `EnemyGroup` (적 생성 위치, Object Pool에서 켜진 적들이 이 하위로 들어옴)
        *   `MinionGroup` (플레이어가 부활시킨 언데드 군단 그룹)
        *   `ItemGroup` (드랍된 경험치 구슬, 체력 포션 등)
    *   `[UI_Root]` (Canvas - ***세부 사항은 4번 항목 참조***)
    *   `EventSystem`

## 3. 클래스 및 인터페이스 구조 (Class Diagram)

1인 개발 시 유지보수와 AI 프롬프트 생성을 용이하게 하기 위한 상속 및 인터페이스 기반 구조입니다.

*   **인터페이스 (Interfaces):**
    *   `IDamageable`: 데미지를 받을 수 있는 객체는 모두 상속. `TakeDamage(int damage)`, `Die()` 구현.
*   **베이스 클래스 (Base Class):**
    *   `UnitBase` (MonoBehaviour, IDamageable 상속)
        *   공통 변수: hp, maxHp, moveSpeed, damage.
        *   공통 함수: `Move()`, `Attack()`, `TakeDamage()`.
*   **자식 클래스 (Derived):**
    *   `PlayerController : UnitBase`
        *   기능: 사용자 Input(WASD)에 따른 이동 역추산. 부활 오라(Trigger) 판정. 죽었을 때 GameManager에 GameOver 호출.
    *   `EnemyBase : UnitBase`
        *   기능: 생성 시 플레이어 위치를 Transform으로 추적(Chase). 죽을 때 ExpManager에 구슬 생성, 확률적으로 `MinionSpawner.Invoke()` 호출.
    *   `MinionBase : UnitBase`
        *   기능: 생성 시 범위 내(OverlapCircle) 가장 가까운 Enemy의 Transform 배열을 검색하여 타겟으로 지정 후 추적/타격.
*   **매니저(Managers) - 싱글톤(Singleton) 패턴 권장:**
    *   `GameManager`: 게임 시간(`gameTime`), 현재 상태(`enum GameState { Play, Pause, GameOver }`) 관리.
    *   `WaveManager`: 시간(`gameTime`)에 따라 `EnemyPool.Get()`을 호출하여 화면 밖 반경(Radius)에 적 무작위 스폰.
    *   `SkillManager`: 플레이어가 레벨업 시 보유한 특성(List<Skill>)의 효과를 `Player`나 `MinionBase`에 실시간으로 브로드캐스트.

## 4. 하이어라키 UI 및 패널 처리 계획

UI는 **하나의 캔버스(`[UI_Root]`) 내에서 Panel 객체들의 Active(켜고 끄기)를 전환**하는 방식을 사용합니다. 이는 씬 전환보다 훨씬 빠르고 가볍습니다.

*   **`[UI_Root]` Canvas 구성 (Z-Index 순, 아래 있을수록 화면 앞쪽):**
    *   **HUD_Panel (항상 활성화):**
        *   상단 중앙: `Text_Timer` (예: 12:34)
        *   상단 좌측: `Image_HPBar` (플레이어 체력 슬라이더 슬라이스)
        *   상단 우측: `Text_MinionCount` (현재 내 군단 수량), `Text_Gold`
        *   하단: `Slider_Exp` (경험치 바, UI 가로 100% 채움)
    *   **LevelUp_Panel (평소 비활성, 경험치 풀 시 활성화):**
        *   `Image_BlurBg` (배경을 어둡게 처리, Time.timeScale = 0 멈춤 전환)
        *   `HorizontalLayoutGroup_Cards`: 각 카드 버튼(프리팹형태) 3개가 무작위 스킬 아이콘, 이름, 설명을 달고 나타남. 클릭 시 스킬 획득 후 패널 비활성화(`Time.timeScale = 1`).
    *   **Pause_Panel (ESC 키 입력 시 활성화):**
        *   현재 플레이어 상세 스탯 텍스트 표시, '계속하기', '포기하고 타이틀로' 버튼.
    *   **GameOver_Panel (플레이어 HP 0일 때 활성화):**
        *   빨간 텍스트 "YOU DIED", 생존 시간 표시, 처치한 적 수, 얻은 재화. '로비로 돌아가기' 버튼.

## 5. 핵심 개발 기능 명세 (What to do list)

코드로 직접 짜야 할, 기획 서류상 가장 중요한 **"기능 세부 목록"**입니다. 이 리스트대로 AI에게 코드를 짜달라고 하면 됩니다.

1.  **무한 맵 스크롤 로직:** 카메라 이동 시 타일맵 배경이 플레이어를 따라와 무한히 맵이 이어지는 것처럼 보이는 기능.
2.  **화면 밖 몬스터 스폰 (Off-screen Spawner):** 메인 카메라 뷰포트(Viewport) 바깥쪽 반지름 R 영역에서 몬스터가 원형으로 둘러싸며 스폰되는 기능.
3.  **몬스터 추적 (Chase & Clustering):** 몬스터가 플레이어를 향해 오되, 서로 겹치지 않게(Collider 밀어내기) 물리 연산을 단순화 하는 로직.
4.  **부활 오라 로직 (Necromancy Aura):**
    *   적이 '사망(Die)' 함수를 호출할 때, 플레이어와의 거리(Distance)가 N 이하인지 판별.
    *   N 이하라면 확률 계산(Random.Range). 확률 부합 시, 해당 적의 좌표 벡터에 `Minion` 오브젝트 풀(Pool)에서 꺼내 활성화.
5.  **미니언 타겟 서칭 (Find Nearest Target):** 부활한 아군이 초당 2~3회만 물리 범위 내의 `Enemy` 태그 오브젝트를 찾고, 거리를 비교해 최단거리 적을 타겟으로 잡는 로직. (매 프레임 연산하지 않는 것이 최적화의 핵심)
6.  **경험치 구슬 물리 및 자석 처리:** 몬스터가 죽은 자리에 보석 생성. 플레이어가 특정 콜라이더 반경 안에 들어오면 보석이 플레이어를 향해 점점 빠르게 날아오도록(Lerp / MoveTowards) 구현.
7.  **오브젝트 풀링 관리자 (Object Pooler):** 몬스터 500마리, 투사체 1,000개 분량을 미리 Awake에 Instantiate 해두고 끄고 켜기만 하는 거대한 리스트/큐 관리 구조 (가비지 컬렉터 부하 방지용).

## 6. 개발 루프(플로우 차트) 요약

1.  씬 로딩 -> `LevelUp_Panel` 포함 모든 패널 비활성화 및 초기 배열, `HUD_Panel` 켬.
2.  `WaveManager` 작동 -> 초당 적 스폰 (초반은 약하고 적게, 후반 갈수록 강하고 많게 함수 곡선 처리).
3.  전투, 적 사망 -> `ExpManager`가 해당 위치에 보석 풀 드랍. `PlayerController` 오라 범위 내라면 적은 아군 `Minion` 풀로 교체 스폰.
4.  경험치 바 도달 -> 시간 정지 -> `LevelUp_Panel` 활성화 -> 카드 3장 로드.
5.  선택 후 `SkillManager` 반영 -> 시간 재개 -> 게임 지속.

## 7. 유니티 프로젝트 폴더 구조 (Project Folder Structure)

1인 개발 및 1달 완성을 목표로 하기 때문에, 방대한 계층 구조보다는 **직관적이고 접근성이 빠른 폴더명(`_` 언더바를 이용한 정렬)**을 사용합니다. 외부 에셋(Asset Store)과 내가 짠 코드/리소스를 철저히 분리하는 것이 핵심입니다.

```text
Assets/
├── _Game/                      # ⭐️ 내가 직접 만든 모든 리소스와 코드는 이 폴더 안에 넣습니다.
│   ├── Animations/             # 플레이어, 적 구동 애니메이션 클립 및 컨트롤러
│   ├── Data/                   # ScriptableObject 타겟 (스킬 테이블, 적 스탯 테이블)
│   ├── Materials/              # 재질 파일
│   ├── Prefabs/                # ⭐️ 가장 중요한 폴더. 씬에 올릴 모든 객체는 프리팹화 해야합니다.
│   │   ├── Characters/         # Player, Enemies, Minions 프리팹
│   │   ├── Environment/        # 배경 타일맵, 텍스처 머티리얼 블록 프리팹
│   │   ├── Props/              # 경험치 보석, 상자, 포션
│   │   ├── UI/                 # 레벨업 스킬카드, 데미지 텍스트 팝업 등
│   │   └── VFX/                # 피격 파티클 등
│   ├── Scenes/                 # TitleScene, GameScene
│   ├── Scripts/                # ⭐️ 내가 작성한 C# 스크립트 모음
│   │   ├── Core/               # 각종 Manager 클래스 (Game, Wave, ObjectPool 등)
│   │   ├── Entities/           # UnitBase, PlayerController, EnemyAI 등
│   │   ├── System/             # 풀링, 데미지 인터페이스 등 기반 로직
│   │   └── UI/                 # UIManager, LevelUpCard 클릭 등 UI 연동 클래스
│   └── Sprites/                # 내가 넣은(또는 AI로 뽑은) 원본 이미지(PNG) (Character, UI, Environment)
│
├── Plugins/                    # DOTween, 기타 외부 툴즈
├── Resources/                  # (가급적 사용 자제 - 특수 상황에서 동적 로드용. Prefabs나 Addressables 권장)
├── Settings/                   # URP(Universal Render Pipeline) 세팅 파일 등
└── [Asset Store 폴더들]        # 에셋 스토어에서 받은 팩 (예: Kenney, 기타 등등)은 최상단에 방치 (절대 수정 금지)
```

## 8. 콘텐츠 (Content) - 몬스터 및 아군 테크트리

**[1] 적 몬스터 10단계 웨이브 테이블 (총 10웨이브 / 15분 기준)**
시간에 따라 난이도가 상승하며, 완전히 새로운 적만 나오는 것이 아니라 **이전 웨이브의 적들이 비율이 섞여서 등장**합니다. (예: 10웨이브에서도 농부가 고기방패 역할로 1~2마리씩 섞여서 스폰됨)
1. **농부 (Lv.1):** 몽둥이 장착. 체력/이속 최하 (가장 만만한 제물)
2. **징집병 (Lv.2):** 쇠스랑 장착. 농민보다 배로 몰려다님

## 13. 고도화 제안 및 QA 체크리스트 (Fun & Stability)

출시까지의 퀄리티를 보장하고 유저 피드백을 반영하기 위한 추가 고도화 항목입니다.

### 13-1. 재미 요소 강화 (Juice & Mechanics)
*   **미니언 시너지 (Synergy):** 특정 조합(예: 해골 20 + 좀비 10) 달성 시 발동되는 '특수 오라' 추가로 빌드 구성의 재미 극대화. (자동 발동)
*   **전술 패시브 (Tactical Passive):** 수동 명령 대신 레벨업 선택지로 군단의 행동 양식을 바꾸는 스킬 추가. (예: 플레이어 주변 보호, 최강 적 우선 타겟팅 등)
*   **타격감 피드백 (Juice):** 적 처치 시 화면 흔들림(Screen Shake), 넉백 연출, 화려한 영혼 흡수 파티클 강화.

### 13-2. QA 및 기술적 안정성 (Bug Prevention)
*   **대규모 최적화 (Entity Optimization):** 500마리 이상의 유닛 처리를 위해 물리 연산 대신 '거리 기반 단순화' 및 '객체 풀링 최적화' 필수.
*   **세이브 데이터 충돌 방지:** 게스트와 구글 세이브 데이터 간의 우선순위 및 충돌 해결 로직 검증.
*   **스테이지 밸런스 시뮬레이션:** 내부 디버그 툴을 통한 밸런스 테스트 수행.

### 13-3. 누락된 시스템 인프라
*   **중앙 사운드 시스템:** SFX/BGM의 가변적 믹싱 및 효과음 전용 사운드 풀러 구비.
*   **디버그/치트 콘솔:** 개발 및 QA 단계에서 스테이지 건너뛰기, 무적 등을 위한 인게임 콘솔 구현.
